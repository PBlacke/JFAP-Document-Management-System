📅 Development Timeline Journal
Project: JFAP Simple Document Management System (DMS) Prototype


🗓️ February 27, 2026 – Project Kickoff & Core Implementation
Started building the prototype from scratch using Flask, SQLite, and Tesseract OCR.
Implemented basic functionality:
    - File upload (images and PDFs) with secure filename handling.
    - OCR text extraction using Tesseract (via pytesseract).
    - Storage of document metadata and extracted text in an SQLite database.
    - Simple search using LIKE queries on the extracted text.
    - Clean, responsive frontend with vanilla HTML/CSS/JavaScript.
    - Set up project structure with app.py, templates/, static/, and uploads/ folders.
    - Initialised Git repository and pushed the first working version to GitHub.
✅ Achieved: A functional DMS that can upload, OCR, store, and search documents.


🗓️ March 2, 2026 – Debugging, Documentation, and Preview Attempt
Focused on refining the existing code:
    - Fixed several typos and syntax errors (e.g., table creation, missing commas, variable name mismatches).
    - Resolved issues with Tesseract path configuration and PDF processing.
    - Corrected the favicon not showing by moving it to the static/ folder and using url_for.
    - Added a comprehensive README to the GitHub repository, detailing installation, usage, and tech stack.
    - Began implementing document preview functionality:
    - Created /preview/<int:doc_id> route to serve images and PDFs in an embedded viewer.
    - Added preview_image.html and preview_pdf.html templates.
Encountered bugs: missing doc_id in template context and SQL query errors (typo in table name, missing commas in tuples).
🐛 Challenges: Preview feature was incomplete due to template variable errors and database query issues.


🗓️ March 3, 2026 – Feature Completion, UI Polish, and Final Debugging
    Enhanced upload interface:
        - Added drag‑and‑drop support to the upload area (files are placed into the file input but not auto‑uploaded – user must click the button, maintaining consistent behaviour).
        - Added Document Type dropdown (PR, PO, Other) and Project text field to the upload form; these are stored in the database.
    Database schema updated:
        - Added doc_type and project columns to the documents table (with error handling for existing columns).
    Redesigned document editing:
        - Replaced the separate edit.html page with a modal popup on the documents page.
        Modal includes:
            - A preview of the document (image thumbnail or PDF icon).
            - Editable filename (without extension), document type dropdown, and project field.
            - A Delete button inside the modal (with confirmation) for convenience.
        - Updated /edit/<int:doc_id> route to handle only POST requests, updating filename, type, and project, and renaming the actual file on disk.
        - Updated /delete/<int:doc_id> route remains POST-only.
    Documents list enhanced:
        - Displays type and project under each filename.
        - Edit button (pencil icon) opens the modal; delete button (trash icon) remains inline for quick deletion (with confirmation).
    Fixed remaining bugs:
        - Corrected JavaScript errors in uploadFile() (moved FormData creation before appending fields).
        - Fixed edit modal population by retrieving data from data-* attributes, ensuring type and project are correctly set.
        - Set dynamic action for both edit and delete forms inside the modal.
        - Resolved SQL parameter tuple issues (commas) in all database calls.
        - Ensured url_for imported everywhere needed.
        - Timestamp formatting now uses standard %Y-%m-%d %H:%M:%S (Windows‑compatible).
    Tested end‑to‑end:
        - Upload with type/project works (both manual and drag‑and‑drop).
        - Search returns results; preview opens correct viewer.
        - Edit modal opens with correct data, saves changes (file renamed, database updated).
        - Delete removes both record and file.
        - All features working smoothly.
✅ Final outcome: A fully functional DMS prototype with upload (with drag‑and‑drop), OCR, search, preview, inline editing modal, and delete, complete with document tagging (type/project) and a polished UI. Well‑documented on GitHub.
